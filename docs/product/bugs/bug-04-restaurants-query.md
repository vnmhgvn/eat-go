## 2. Description

### Summary

When navigating to `/sessions/new`, the application crashes with a PostgreSQL error `Failed query: select ... from "restaurants" where "restaurants"."created_by" params: `. This happens because the Drizzle ORM `where()` clauses were malformed, passing column references directly instead of valid condition expressions like `eq()`.

### Steps to Reproduce

```
1. Log in.
2. Go to Dashboard and click "Tạo phiên mới" (navigates to `/sessions/new`).
3. Observe: 500 server error complaining about malformed `where` clause.
```

### Expected Behavior

The page should load correctly and display the unified list of global restaurants and the user's personal restaurants in the `<select>` dropdown.

### Actual Behavior

The server crashes during rendering because the SQL query generated by Drizzle is invalid (`where "restaurants"."created_by"` without an `=` condition).

## 5. Root Cause Analysis

### Root Cause
The Drizzle `where()` clause requires a full expression like `eq(column, value)`. In `src/app/(app)/sessions/new/page.tsx`, the queries were hastily written as `where(restaurants.isGlobal)` and `where(restaurants.createdBy)`. These evaluate to SQL fragments like `WHERE is_global` and `WHERE created_by`, which are structurally incomplete or syntactically invalid, causing Postgres to throw an error.

### Affected Files
| File | Issue |
| ---- | ----- |
| `src/app/(app)/sessions/new/page.tsx` | Malformed Drizzle `where()` clauses on lines 13 and 14. |

---

## 6. Resolution

### Fix Description
1. Imported `eq` and `or` from `drizzle-orm`.
2. Refactored the data fetch to a single efficient query combining both conditions using `or()`: `where(or(eq(restaurants.isGlobal, true), eq(restaurants.createdBy, user.id)))`. This correctly generates `WHERE is_global = true OR created_by = '...'`.
3. Simplified the data transformation logic since the DB now correctly filters exactly the needed records.
